I’ll provide a fully structured, self-contained index.html file that includes HTML, plain CSS, and JavaScript for a website deployed on GitHub Pages. The site will use a data.csv file in your GitHub repository as a database, supporting CRUD (Create, Read, Update, Delete) operations via the GitHub API. I’ll also include a sample data.csv with pre-stored data, clear instructions on what to replace (e.g., GitHub username, repository name, and Personal Access Token), and ensure the data is displayed as plain text in a table. The goal is to let you test if this works by deploying it to your GitHub repository.

Solution Overview

  • Repository Structure: Your GitHub repository contains:
    • index.html: The main file with HTML, CSS, and JavaScript.
    • data.csv: The CSV file acting as the database.
  • Functionality:
    • Read: Fetch and display the CSV data in a table.
    • Create: Add new data via a form.
    • Update: Edit existing data.
    • Delete: Remove data entries.
  • GitHub API: Used to read and update data.csv in the repository.
  • Deployment: Hosted on GitHub Pages, with data persisting across devices via the repository.

<body>
    <h1>CSV Database Website</h1>
    <div class="form-container">
        <h2>Add/Edit Data</h2>
        <input type="hidden" id="editIndex" value="">
        <input type="text" id="name" placeholder="Enter name" required>
        <input type="text" id="message" placeholder="Enter message" required>
        <button onclick="submitData()">Submit</button>
    </div>
    <h2>Data Table</h2>
    <div id="error" class="error"></div>
    <table>
        <thead>
            <tr>
                <th>Name</th>
                <th>Message</th>
                <th>Actions</th>
            </tr>
        </thead>
        <tbody id="tableBody"></tbody>
    </table>

    <script>
        const repoOwner = "YOUR_GITHUB_USERNAME"; // Replace with your GitHub username
        const repoName = "YOUR_REPO_NAME"; // Replace with your repository name
        const filePath = "data.csv"; // Path to CSV in repo
        const token = "YOUR_PERSONAL_ACCESS_TOKEN"; // Replace with your GitHub Personal Access Token

        let csvData = [];

        // Display error message
        function showError(message) {
            document.getElementById("error").innerText = message;
            setTimeout(() => document.getElementById("error").innerText = "", 5000);
        }

        // Fetch and display CSV data
        async function loadData() {
            try {
                const response = await fetch(`https://raw.githubusercontent.com/${repoOwner}/${repoName}/main/${filePath}`);
                if (!response.ok) throw new Error("Failed to fetch CSV");
                const csvText = await response.text();
                csvData = csvText.trim().split("\n").map(row => row.split(","));
                const tableBody = document.getElementById("tableBody");
                tableBody.innerHTML = "";
                // Skip header row
                for (let i = 1; i < csvData.length; i++) {
                    const row = csvData[i];
                    const tr = document.createElement("tr");
                    tr.innerHTML = `
                        <td>${row[0]}</td>
                        <td>${row[1]}</td>
                        <td class="action-buttons">
                            <button class="edit-btn" onclick="editData(${i})">Edit</button>
                            <button class="delete-btn" onclick="deleteData(${i})">Delete</button>
                        </td>`;
                    tableBody.appendChild(tr);
                }
            } catch (error) {
                showError("Error loading data: " + error.message);
            }
        }

        // Get current CSV file details (content and SHA)
        async function getFileDetails() {
            const response = await fetch(`https://api.github.com/repos/${repoOwner}/${repoName}/contents/${filePath}`, {
                headers: {
                    Authorization: `Bearer ${token}`,
                    Accept: "application/vnd.github.v3+json"
                }
            });
            if (!response.ok) throw new Error("Failed to fetch file details");
            return await response.json();
        }

        // Update CSV file via GitHub API
        async function updateFile(newContent) {
            try {
                const fileDetails = await getFileDetails();
                const encodedContent = btoa(newContent); // Encode to base64
                const response = await fetch(`https://api.github.com/repos/${repoOwner}/${repoName}/contents/${filePath}`, {
                    method: "PUT",
                    headers: {
                        Authorization: `Bearer ${token}`,
                        Accept: "application/vnd.github.v3+json"
                    },
                    body: JSON.stringify({
                        message: "Update data.csv",
                        content: encodedContent,
                        sha: fileDetails.sha
                    })
                });
                if (!response.ok) throw new Error("Failed to update file");
                await loadData(); // Refresh data
            } catch (error) {
                showError("Error updating data: " + error.message);
            }
        }

        // Submit data (Create or Update)
        async function submitData() {
            const editIndex = document.getElementById("editIndex").value;
            const name = document.getElementById("name").value.trim();
            const message = document.getElementById("message").value.trim();
            if (!name || !message) {
                showError("Please fill in both fields");
                return;
            }
            if (editIndex === "") {
                // Create: Append new row
                csvData.push([name, message]);
            } else {
                // Update: Modify existing row
                csvData[parseInt(editIndex)] = [name, message];
            }
            const newContent = csvData.map(row => row.join(",")).join("\n");
            await updateFile(newContent);
            document.getElementById("editIndex").value = "";
            document.getElementById("name").value = "";
            document.getElementById("message").value = "";
        }

        // Edit data
        function editData(index) {
            document.getElementById("editIndex").value = index;
            document.getElementById("name").value = csvData[index][0];
            document.getElementById("message").value = csvData[index][1];
        }

        // Delete data
        async function deleteData(index) {
            if (confirm("Are you sure you want to delete this entry?")) {
                csvData.splice(index, 1); // Remove row
                const newContent = csvData.map(row => row.join(",")).join("\n");
                await updateFile(newContent);
            }
        }

        // Load data on page load
        loadData();
    </script>

What to Replace

In the <script> section of index.html, replace the following placeholders with your actual values:

  1. YOUR_GITHUB_USERNAME:

    • Replace with your GitHub username (e.g., johnsmith if your GitHub profile is github.com/johnsmith).
    • Example: const repoOwner = "johnsmith";

  2. YOUR_REPO_NAME:

    • Replace with the name of your GitHub repository (e.g., my-website if your repo is github.com/johnsmith/my-website).
    • Example: const repoName = "my-website";

  3. YOUR_PERSONAL_ACCESS_TOKEN:

    • Generate a GitHub Personal Access Token:
      • Go to GitHub > Settings > Developer settings > Personal access tokens > Tokens (classic).
      • Click Generate new token.
      • Select the repo scope.
      • Copy the token (e.g., ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx).
    • Replace the placeholder with this token.
    • Example: const token = "ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx";
    • Warning: Do not share this token publicly or commit it to your repository. For testing, you can include it in the code, but for production, use a backend to handle API requests securely (see security notes below).

How to Deploy and Test

  1. Set Up Your Repository:

    • Ensure your GitHub repository has:
      • data.csv with the sample content provided above.
      • index.html with the code provided, after replacing the placeholders.

How It Works

  • Read: On page load, JavaScript fetches data.csv from https://raw.githubusercontent.com/... and displays it in a table.
  • Create: Submitting the form appends a new row to csvData, updates data.csv via the GitHub API, and refreshes the table.
  • Update: Clicking Edit populates the form with the row’s data. Submitting updates the row in csvData and the CSV file.
  • Delete: Clicking Delete removes the row from csvData and updates the CSV file.
  • Persistence: The CSV file is stored in the GitHub repository, so updates are persisted and reflected across devices on refresh.

Security Notes

  • Personal Access Token: Including the token in client-side JavaScript is insecure, as anyone can access it via browser developer tools. For testing, this is fine, but for production:
    • Set up a simple backend (e.g., a Node.js server or serverless function on Vercel/Netlify) to handle GitHub API requests.
    • Store the token securely in the backend (e.g., as an environment variable).
    • Expose an endpoint (e.g., /update-csv) that the frontend calls.
  • CSV Injection: The code assumes simple text input. For production, validate inputs to prevent CSV injection (e.g., reject values starting with =, +, -, @).
  • Rate Limits: The GitHub API has a 5,000 requests/hour limit for authenticated requests. For high traffic, cache the CSV content in the frontend or use a backend to reduce API calls.

Troubleshooting

  • Table is empty: Ensure data.csv exists in the repository and has the correct format. Check the browser console for errors.
  • API errors: Verify the Personal Access Token has the repo scope and is correct. Check rate limits.
  • Data not updating: Ensure the repository name and username are correct. GitHub Pages may take a few seconds to reflect changes.
